Windows 10 14316 bash安装错误或启动错误

发表于 2016-04-11   |   分类于 Systems ， Windows   |  
启动功能
遇到问题

启动功能

14316测试版中已经提供bash组件，默认关闭的，启用的方法是先选中“开始->设置->更新和安全->针对开发人员->开发人员模式”，然后按 Win+X,F 打开“程序和功能”，点击左侧的“启用或关闭Windows功能”，然后找到并勾选 Windows Subsystem for Linux (Beta) ，确定后会提示重启启动系统。

系统重启完成后，按 Win+X,A 打开命令提示符（或者按 Win+R 打开运行对话框），输入 bash 回车，第一次运行会从网上下载Ubuntu linux组件，按提示输入y。


1.  `-- Beta feature --`
2.  `This will install Ubuntu on Windows, distributed by Canonical`
3.  `and licensed under its terms available here:`
4.  `https://aka.ms/uowterms`
5.  `Type "y" to continue: y`
6.  `Downloading from the Windows Store... 100%`
7.  `Extracting filesystem, this will take a few minutes...`
8.  `????! ???????...`
9.  `root@localhost:~#`

查看版本信息和分区映射：


1.  `root@localhost:~# uname -a`
2.  `Linux localhost 3.4.0+ #1 PREEMPT Thu Aug 1 17:06:05 CST 2013 x86_64 x86_64 x86_64 GNU/Linux`
3.  `root@localhost:~# cat /etc/issue`
4.  `Ubuntu 14.04.4 LTS n l`
5.  `root@localhost:~# ls -al /mnt`
6.  `total 24`
7.  `drwxrwxr-x 2 root 1000 0 Apr  7 07:22 .`
8.  `drwxrwxr-x 2 root root 0 Apr  7 07:22 ..`
9.  `drwxrwxrwx 2 root root 0 Apr  7 06:58 c`
10.  `drwxrwxrwx 2 root root 0 Apr  7 04:59 d`
11.  `root@localhost:~#`

遇到问题

注意是root用户，/dev 下也没有很多的设备，输入 exit 可退出bash。

/root 目录对应Windows系统的 %localappdata%lxssroot 文件夹（注意，不要删除lxss目录，bash会失败）。

如果不幸删除了lxss或者其他原因 bash 启动失败。可以在CMD里用下面两条命令卸载再重新安装：


1.  `echo y|lxrun /uninstall `
2.  `echo y|lxrun /install`

使用命令 apt-get 安装软件前建议先在 /etc/apt/sources.list 文件前端添加国内的镜像服务器， http://mirrors.163.com/ubuntu 是163的镜像源：


1.  `deb http://mirrors.163.com/ubuntu trusty main restricted universe multiverse `
2.  `deb http://mirrors.163.com/ubuntu trusty-upadates main restricted universe multiverse `
3.  `deb http://mirrors.163.com/ubuntu trusty-security main restricted universe multiverse`

然后用 apt-get update 命令更新源（163的更新源最后会出现一些校验警告，可以忽略）。

还有一种方式，在刚刚安装好bash后，用支持LF换行的文本编辑器修改sources.list文件（下面的顺序中C:>表示CMD下，#表示在bash下）


1.  `C:> echo y|lxrun /uninstall           // 卸载linux子系统`
2.  `C:> rmdir /s /q %localappdata%lxss   // 删除lxss文件夹`
3.  `C:> echo y|lxrun /install             // 安装linux子系统，如果网络不好可以多试几次。`
4.  `C:> bash`
5.  `# touch /etc/apt/sources.list`
6.  `# exit`
7.  `C:> notepad2 %localappdata%Lxssrootfsetcaptsources.list          // 用notepad2文本编辑器粘贴163镜像`
8.9.  `----------以下为文件内容-----------`
10.  `# %localappdata%Lxssrootfsetcaptsources.list`
11.  `# 163`
12.  `deb http://mirrors.163.com/ubuntu trusty main restricted`
13.  `deb-src http://mirrors.163.com/ubuntu trusty main restricted`
14.  `deb http://mirrors.163.com/ubuntu trusty-security main restricted`
15.  `deb-src http://mirrors.163.com/ubuntu trusty-security main restricted`
16.  `deb http://mirrors.163.com/ubuntu trusty-updates main restricted`
17.  `deb-src http://mirrors.163.com/ubuntu trusty-updates main restricted`
18.  `# ubuntu`
19.  `#deb http://us.archive.ubuntu.com/ubuntu trusty main restricted`
20.  `#deb-src http://us.archive.ubuntu.com/ubuntu trusty main restricted`
21.  `#deb http://security.ubuntu.com/ubuntu trusty-security main restricted`
22.  `#deb-src http://security.ubuntu.com/ubuntu trusty-security main restricted`
23.  `#deb http://us.archive.ubuntu.com/ubuntu trusty-updates main restricted`
24.  `#deb-src http://us.archive.ubuntu.com/ubuntu trusty-updates main restricted`
25.  `----------文件内容结束----------`
26.27.  `C:> lxrun /update             //更新apt源`

安装 git：


1.  `apt-get --assume-yes install git`
